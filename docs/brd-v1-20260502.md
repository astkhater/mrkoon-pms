# Business Requirements Document (BRD) — Mrkoon OKR/KPI/Appraisal Web App
Version: 1
Date: 2026-05-02
Owner: Build session
Source: handover-note v1 (2026-05-01)

## 1. Context
Mrkoon currently runs OKR/KPI/Appraisal/Commission across 28 Excel files spread over `mrkoon-chro/`, `shared/hr/`, `mrkoon-ops/`, and `shared/commercial/`. The 2026-05-01 alignment review identified four conceptual layers (config, definitions, calculations, tracking), 22 fixes applied, 5 empty shells deprecated, and Option B selected: consolidate into a single platform — this app.

## 2. Problem statement
- **Hard-code-by-copy**: 6+ files duplicate the same canonical values (300, 30M, 0.05, 25, 4, 6). Updating any one creates drift.
- **One-way SOP↔KPI linkage**: KPI Definitions reference SOPs but reverse query needs manual file-by-file search.
- **Two-format duplication**: appraisal-annual.xlsx + appraisal-form-annual.docx carry parallel content.
- **Manual quarterly→annual aggregation**: Q1–Q4 transcribed by hand into annual aggregator.
- **Per-VM, loading, merchant DB activation tracking has no data source** (KPI-033..036 added this session, weight 0 until CHRO rebalance).
- **5 empty shells**: shared trackers that claim a tracking role they don't fulfil.
- **Performance Gates** as named individuals exposed in templates.
- **DEC codes** leaking into user-visible labels.

## 3. Objectives
- Single platform replacing 28 Excel files for OKR, KPI, appraisal, bonus/commission, and SOP reference.
- Bidirectional SOP↔KPI navigation, both directions O(1).
- Auto-aggregate Q1–Q4 → annual appraisal.
- Single source of truth: every value referenced, never duplicated.
- Bilingual EN/AR with full RTL support.
- Role-aware access: no role sees data above permission level.
- Full audit trail: every entry, every change, every approval.
- CRM commission sync (existing webapp), ERP integration — both dormant until credentials provided.
- Standalone from day 1 — works fully without CRM or ERP.

## 5. Scope

### In scope
- OKR module: company/dept/individual cascade, period management, progress tracking
- KPI module: 36 KPIs seeded, formula engine, traffic-light dashboards, SOP cross-ref
- Appraisal module: monthly check-in, quarterly review, yearly appraisal, calibration
- Bonus/commission engine: 8 schemes (BD/AM/VM/Ops/HussTL/Onb/QrtlOpEx/Annual)
- SOP reference: 10 SOPs linked bidirectionally to KPIs
- Audit trail across all modules
- Role-based dashboards (Employee/Manager/Dept Head/HR/Finance/C-Level/Admin)
- Bilingual EN/AR with RTL toggle
- Desktop-first, mobile responsive
- CRM Edge Function (dormant)
- ERP Edge Function (dormant)

### Out of scope
- SOP-009/010 content drafting (SOP session)
- Salary master (lives in payroll/ERP)
- Investor/equity tracking (per confidentiality rule)
- DEC code governance (CHRO/CoS)
- KSA SaaS-tier billing (separate product)
- Public-facing portal (this is internal only)

## 6. Success metrics
- 28 Excel files retired and superseded by platform within 90 days of go-live.
- Zero hard-coded values in calculation paths (every value traceable to `config.*`).
- Q1–Q4 → annual aggregation executes automatically with zero manual transcription.
- Bidirectional SOP↔KPI lookup returns < 200 ms.
- Role-isolation test passes: zero role sees data above its permission level.
- Bilingual coverage: 100% of UI strings in `lang/ar.js` and `lang/en.js`, zero hardcoded strings in components.
- Appraisal cycle completion rate trackable to ≥95% within first full cycle.

## 7. Constraints
- Must reuse Mrkoon CRM v15 patterns (auth, bilingual toggle, RLS structure, dormant Edge Function pattern).
- Internal-only — Netlify access restricted.
- Staging environment required before production launch (per build-standards).
- Cannot execute Supabase deployment without Khater present.
- 7 pre-build documents required before code (BRD/TRD/SDD/User Stories/Use Cases/Acceptance Criteria/Test Cases).

## 8. Dependencies
- Mrkoon CRM v15 webapp (commission sync target)
- Future ERP system (payroll/employee master)

## 9. Risks
| Risk | Likelihood | Impact | Mitigation |
|---|---|---|---|
| Formula misinterpretation between Excel and platform | Medium | High | Phase 6 verifies every calc against handover note examples |
| Role isolation gap | Low | Critical | RLS at DB level + Phase 1 isolation tests required to pass before any other phase |
| Bilingual drift | Medium | Medium | All text in lang files; lint rule no hardcoded strings |
| Excel users resist platform adoption | Medium | High | Read-only Excel viewer phase; 90-day parallel run |
| CRM credentials delayed | High | Low | Dormant Edge Function pattern — app fully functional without CRM |
| Operations TL KPI sum 0.50 / Tech 1.35 surfaces as scoring bug | Low | Medium | Seed current values; flag in admin UI; CHRO rebalance via config not code |
